31. What is Oozie?

Oozie is a workflow scheduler for Hadoop jobs.

It can schedule:

MapReduce jobs
Hive jobs
Pig jobs
Sqoop jobs
32. What is HBase?

HBase is a NoSQL database built on top of HDFS.

It is used for real-time read/write access to large datasets.

33. Difference between Hadoop MapReduce and Spark?
Hadoop MapReduce	Apache Spark
Disk-based processing	In-memory processing
Slower	Faster
Good for batch jobs	Good for batch, streaming, ML
More boilerplate code	Easier APIs
Uses MapReduce engine	Uses DAG engine

34. Can Spark run on Hadoop?

Yes. Spark can run on Hadoop using:

HDFS for storage
YARN for resource management

35. What happens if a DataNode fails?

The NameNode detects failure through missing heartbeats.

Then:

It marks the DataNode as dead.
It identifies under-replicated blocks.
It creates new replicas on healthy DataNodes.

36. What happens if one block is corrupted?

HDFS detects corruption using checksums.

It reads another replica of the block and creates a fresh copy.

37. How do you improve Hadoop job performance?

Ways to improve performance:

Use Combiner when possible
Avoid small files
Use proper file formats like Parquet or ORC
Tune number of mappers and reducers
Use compression
Use partitioning and bucketing in Hive
Avoid data skew

38. What is data skew?

Data skew happens when some keys have much more data than others.

Example:

(city = New York) has millions of records
(city = Auburn) has thousands

One reducer gets overloaded while others finish quickly.

39. How do you handle data skew?

Possible solutions:

Use custom partitioner
Add salting to keys
Increase reducers
Pre-aggregate data
Filter or split heavy keys separately

40. What is speculative execution?

Speculative execution runs duplicate copies of slow tasks on other nodes.

Whichever copy finishes first is accepted.

This helps when a task is slow because of hardware or network issues.
